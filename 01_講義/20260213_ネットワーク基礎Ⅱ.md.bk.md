# はじめに
- ルータ
    - 複数ネットワークを接続する。</br>
    目的のコンピュータに対し直接通信できるため、効率がよい。

- スイッチ
    - ネットワーク内の複数コンピュータ接続。

- ハブ
    - ルータと役割が変わらないが、効率が悪い。</br>
    接続しているずべてのコンピュータに通信を行うため。

- OSI参照モデル
    - コンピュータ同士が通信するときにどのように、情報をやりとり捨華を決めるルールブックのようなもの。

- プロトコル
    - 通信をする時のルール。

# 物理層
レイヤー1
ハードウェアのドラブルや通信環境の問題が発生したときは物理層に問題がある可能性が高くなる。

- 物理層で使われる規格
・IEEE 802.3
・bluetooth
・USB

- ネットワークケーブルの主な種類
ツイストペアケーブル
光ファイバーケーブル

- Ethernet
通信規格を揃えるのは、通信が弱くなるのを防ぐためである。


- トポロジー
スター型トポロジ
ツリー型トポロジ
バス型トポロジ
リング型トポロジ
メッシュ型トポロジ
    フルメッシュ型トポロジ：全てのコンピュータに経路を繋ぐため、拡張コストが高い。
    部分メッシュ型トポロジ：必要な箇所だけ経路を繋ぐため、拡張コストが抑えられる。



# データリンク層
レイヤー2

- データ層で扱うプロトコル

Point-to-Point protocol(PPP)
Token Ring
Frame Relay


- パケット通信とは
パケットに番号を振り、正しい順番でデータを受け取ることができる。
暗号化アルゴリズムが使用される。代表的なものは、AES、RSA、DHなどがある

# トランスポート層
レイヤー4

end-to-endの通信をサポートするため、ポート番号を定義する。



- トランスポート層で扱うプロトコル
・TCP
・UDP

- 通信の確立、維持、終了、およびエラー処理
1. 通信の確立: SYC→SYN ACK→SCK
2. 通信の維持
3. 通信の終了: FIN→FIN ACK→ACK
4. エラー処理

9. ロードバランサー



# 通信制御
コンピュータ同士が通信するとき、データを送受信する通信制御をNW通信制御と呼ぶ。
(例) 通信の弱いエリアでは動画の画質を落として、確実に通信を届ける。
帯域の制御とは、
フラッディング：通信が溢れること。


### フロー制御
ネットワーク上のトラフィックを制御することで、通信速度や帯域幅の均等な仕様を実現するための技術です。
- コネクション制御

- 輻輳(ふくそう)制御


### QoS
Quality of Serviceの略

- 重要なアプリケーションやサービスに対して、帯域幅や遅延などの性能を保証することでそれらのアプリケーションやサービスを優先する
- ネットワーク上のトラフィックを制御し、フラッディングなどの問題を防止
- ネットワーク上のトラフィックを割り当てるために聖域幅や遅延などの性能要件を指定する。
