# メールサーバ
## メールサーバとは
以下のサーバに分かれている。
```bash
1. メール送信や配送を行うSMTPサーバ
2. ユーザがメールを受信するための機能を提供するPOPサーバまたはIMAPサーバ
```
</br>

## メールアドレス
メールアドレスの仕組みは以下に分けられる。
```bash
1. ユーザ名@ドメイン名（例：ユーザ名@nobita.com）
2. ユーザ名@ホスト名（例：ユーザ名@www.nobita.com）
```
</br>


## `SMTPサーバ`
メールを送り先に移動させるためのサーバ。</br></br></br>


## メールの保存形式
#### Mailbox形式
一つのファイルメールが追記される形式。</br>
ファイルが削除されると全てのメールが削除されてしまう。</br>
**%平文%{レッド}** で保存されていくため、暗号化されていない。</br>
同じディレクトリに他人のメールファイルも存在するため、権限が付けられず見られてしまう。</br>

#### Maildir形式
普段自分たちが使用している形式。</br>
各アカウントごとにディレクトリが切られていて、そのディレクトリごとにファイルが作成される。</br>
リスクヘッジのためにこちらが採用されている。</br></br></br>


## メールの読み出し
メールを呼び出すために、以下プロトコルを使用する必要がある。</br>

#### POP
現在は、**%POP3%{レッド}** が使われている。</br>
メールサーバに受信BOXに届いた後、ユーザのPCにダウンロードしてPC上で管理する。</br>
サーバにメールが蓄積されないため、サーバ側の容量を気にしなくていいのが特徴。</br>
(受信するとサーバからメールが消える仕様)</br>
**`例）通信会社のキャリアメール`**</br>

#### IMAP
現在は、**%IMAP4%{レッド}** が使われている。</br>
受信BOXを参照して、PC上に表示させるだけ。</br>
**`例）Gmail`**</br></br></br>
