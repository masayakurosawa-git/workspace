# アプリケーションサーバの役割
作成したプログラムを動作させるサーバ。
Tomcatのインストールにjavaも必要。


# Tomcatとは
javaを動かすためのAPサーバ。

javaサーブレット
コンテンツを動的に生成したり、データ処理を行うためにjavaで作成されたプログラム、仕様のこと。

JSP
html内にJavaのコードを埋め込んでおき、部分的にwebページを生成している。



# Apache と Tomcat の違い
webサーバとして使用することは可能。
インターメットからのリクエストはApacheやnginxで受付け、javaで動的な処理が必要なページ処理はバックエンドとなるTomcatで実行するのが一般的です。

Tomcatは、Apache/Nginxほど細かな設定ができない。
Tomcatは、Apache/Nginxでしか提供していない機能に対する多くなサイト要望に答えられない可能性あり。

# プロキシサーバ
ユーザがwebサーバにアクセスするときに、仲介するサーバ。
クライアント - プロキシサーバ - webサーバ


- 自分から見た時の自分を表すIPアドレス
127.0.0.1:8080
{自分から見た時のマイIPアドレス}:{tomcatのポート番号}


# ApacheとTomcat間で通信を行うための設定
1. Apacheにモジュール「mod_proxy」をインストールする。


2. Apacheに以下設定を入れる。
ProxyPass /test/ http://127.0.0.1:8080/
※/test/にアクセスがあれば、tomcatにリダイレクトするようになる。
