## SRX
特定の攻撃に対する防御を設定することができる。

## ファイヤーウォール
外部ネットワークと内部ネットワークの間に立つセキュリティシステム
外部から内部への通信を監視し、許可していない通信をブロックする

### 防御方法
- パケットフィルタリング
- アプリケーションレベルゲートウェイ


## Juniper
SRXをAWS上に構築し、使用していく。

### Juniper とは
独自のOS「junos OS」で動作する。
これまでのネットワーク機器は設定変更が即時反映されていたため、入力ミスなどのヒューマンエラーがネットワーク障害に直結していた。
junos OSでは設定変更後に、適用するためのアクションが挟まれるため、安全に反映を行うことができる。


### CLI仕様
- Shellモード「%」

```
↑exit  ↓cli
```

- Operationalモード「>」

```
↑exit  ↓configure
```

- Configurationモード「#」











### コマンド紹介
- Operationモード
    - show
        - システム
    - monitor
    - request


- Configurationモード
    - commit
        - 変更した設定をシステムに反映させる。
    - commit check
        - コミットが成功するかが確認できる。
    - rollback
        - 以前の設定に戻す。
    - set
        - 設定を追加する。commitして設定反映。
    - delete
        - 設定を削除する。commitして設定反映。


### 実践
インターフェース設定
各インターフェースのIPアドレスを設定する。


ポリシー設定
信頼できるゾーン：内部→外部
信頼できないゾーン：外部→内部


NAT設定
プライベートIPをグローバルIPと紐付け、仮装IPに向けて投げられたリクエストを内部サーバにルーティングするための設定を行う。


スクリーン設定
攻撃者からの不正な通信を防ぐためのスクリーンオプションの設定を行う。
同一IPからの連続的な通信を防ぐ。
